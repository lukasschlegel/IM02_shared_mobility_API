# Shared Mobility API Projekt von Tonia Moser, Ella Ricci und Lukas Schlegel

## Kurzbeschreibung des Projekts
Für unsere Projektarbeit haben wir eine Website entworfen und entwickelt, die es Nutzern ermöglicht, verfügbare E-Scooter in ihrer Umgebung zu suchen und zu finden. Die Seite ist benutzerfreundlich gestaltet und bietet eine interaktive Karte zur Anzeige der Scooter-Standorte. Mithilfe von JavaScript haben wir die API von Shared Mobility integriert, um Echtzeitdaten der Scooter bereitzustellen. Dadurch können Nutzer die nächstgelegenen Scooter finden, deren Verfügbarkeit überprüfen und direkt nutzen. Dieses Shared Mobility API Projekt wurde von Tonia Moser, Lukas Schlegel und Ella Ricci entwickelt.
Funktionen der Seite

### Unsere Website bietet einige Funktionen, um die Nutzererfahrung zu optimieren:
- Interaktive Karte: Die Hauptfunktion der Website ist eine interaktive Karte, die alle verfügbaren E-Scooter in der Umgebung anzeigt. Nutzer können auf die einzelnen Scooter klicken, um weitere Informationen zu erhalten.
- Suchfunktion: Mit der Suchfunktion können Nutzer gezielt nach Scootern in ihrer Nähe suchen.
- Echtzeitdaten: Die Integration der Shared Mobility API stellt sicher, dass die angezeigten Daten immer aktuell sind.
- Nutzerfreundliches Design: Die Seite ist so gestaltet, dass sie intuitiv und einfach zu bedienen ist.

### Styling der Seite
Das Design der Seite ist sehr modern und frisch gehalten, um vor allem eine junge Zielgruppe anzusprechen. Als wiederkehrendes Element wird immer wieder der grüne Blitz eingesezt um die dynamik des Unternehmens wiederzuspiegeln. Ansonsten ist das Design sehr schlicht gehalten, da die Usability im Vordergrund liegen soll.

### Beschreibung der Seite 
Die Startseite der Website begrüsst die Nutzer mit einer klar strukturierten Übersicht und einer interaktiven Karte. Auf dieser Karte werden die Standorte aller verfügbaren E-Scooter in Echtzeit angezeigt. Durch das Klicken auf einen Scooter erhalten die Nutzer die Möglichkeit direkt auf die App des Anbieters abzuspringen und so den E-Scooter zu verwenden. Die Suchfunktion ermöglicht es den Nutzern, spezifische Standorte einzugeben und sofort die nächstgelegenen Scooter zu finden.

## HTML Learnings/Schwierigkeiten
Wir haben HTML verwendet um die Struktur unserer Website zu erstellen. Da sich unser Projekt vor allem durch die interaktive Karte auszeichnet, liegt unser Fokus darauf. Die HTML-Struktu ist daher sehr einfach gehalten und daher hatten wir auch keine Schwierigkeiten. Auch Learning hatten wir keine neuen, da wir mit dem letztjährigen Projekt schon mit gutem HTML-Wissen gewappnet waren.

## CSS Learnings/Schwierigkeiten
Wir haben CSS verwendet um die ganze Seite zu stylen und um dem User über CSS-Animationen ein gutes Nutzererlebnis zu bieten. Unsere Schwierigkeiten im CSS waren vor allem herauszufinden wie wir die von Java Script erstellten Elemente ansprechen können. Ausserdem haben wir einige Zeit gebraucht um zu verstehen, wie wir unserem Konzept entsprechend den Header gestalten können, so dass er aber immernoch responsive reagiert. Daraus konnten wir dann unser Learning ziehen, dass das Bild sich dem Inhalt anpassen soll und nicht umgekehrt.

## Javascript Learnings/Schwierigkeiten
Wir haben die Programmiersprache JavaScript verwendet, um auf die vorgegebene API von Shared Mobility zuzugreifen und die gewonnenen Daten in einer interaktiven Karte anzuzeigen. Für diese interaktive Karte haben wir auf die Dienste von MapBox zurückgegriffen und mithilfe eines kostenlosen Access Tokens eine Basic-Karte für unsere Webseite erstellt, die wir im Verlauf des Projekts immer weiter verfeinert und angepasst haben (Styling, Startpunkt, Suchfunktion, Geolocator). Durch die Integration der Daten der API in die Map konnten wir nun Scooter von verschiedenen Anbietern anzeigen lassen. 

- Learnings: Ein gutes Learning war die richtige Verwendung von "Dot Notations" und die korrekte Integration von Objekten in "Template Strings", sowie die praktische Anwendung von Funktionen und Arrays.
- Schwierigkeiten: Die größte Herausforderung bestand darin, die Scooter immer nur an dem Ort anzuzeigen, der im Suchfeld eingegeben wurde. Es war schwierig, die Koordinateninformationen der Suche zu extrahieren und in das JavaScript zu integrieren.
